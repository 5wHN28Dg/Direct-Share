## Contributing

Contributions are welcome, but please understand the nature of this project.

### General rules

- The project direction is opinionated and intentionally minimal
- Not all contributions will be accepted
- Simplicity and maintainability are prioritized over features

### Pull Requests

PRs are preferred over feature requests.

Before submitting a PR:

- Keep it minimal and focused
- Avoid adding complexity or bloat
- Follow the existing style and structure

PRs may be rejected if they:

- Add features that don't align with the project's goals
- Increase complexity without strong justification
- Solve problems I don't personally have

### Issues

- Bug reports are appreciated
- Feature requests are low priority and may be ignored

### Support

There is no guaranteed support.

If you need help, you are encouraged to:
- Read the code
- Debug on your own
- Submit a fix if necessary
